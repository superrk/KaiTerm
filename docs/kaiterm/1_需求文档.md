# 功能需求文档: KaiTerm

## 0. 功能简介 (Overview)

- **一句话描述**：为开发者和运维人员提供 SSH 终端与 SFTP 文件管理一体化桌面工具，支持多服务器连接管理与 Windows 本地终端。
- **所属模块**：核心功能
- **功能类型**：新增功能

## 1. 背景与目标 (Context & Goals)

- **背景**：现有工具要么是纯 SSH 终端（如 PuTTY），要么是纯 SFTP 客户端（如 FileZilla），两者切换效率低，会话不共享。开发者需要一款集成度高、体验流畅的 SSH+SFTP 一体化工具。
- **目标**：提供 SSH 终端与 SFTP 文件管理共享会话的能力，支持多 Tab 多服务器并发连接，并内置 Windows 本地终端能力。

## 3. 验收标准 (Acceptance Criteria)

### US-001: 连接管理
- [ ] 支持新建连接，输入：连接名称、主机地址、端口（默认 22）、用户名
- [ ] 支持三种认证方式切换：密码、私钥文件、SSH Agent
  - 密码模式：输入密码，密码不显示明文
  - 私钥模式：选择本地私钥文件路径，支持常用格式（OpenSSH、PuTTY .ppk）
  - SSH Agent 模式：自动读取系统 SSH Agent 中的密钥
- [ ] 连接信息（含加密后的凭据）可保存到本地，重启后连接列表保留
- [ ] 密码/私钥敏感信息使用平台级加密存储（Windows DPAPI）
- [ ] 支持从已保存连接列表中快速选择并连接
- [x] 异常场景：连接超时（默认 15s）时提示"连接超时，请检查主机地址和端口"
- [x] 异常场景：认证失败时提示"认证失败，请检查用户名/密码/密钥"
- [x] 异常场景：主机不可达时提示"无法连接到主机，请检查网络连接"
- [ ] 边界情况：端口号输入非数字或超出范围（1-65535）时校验提示

### US-002: SSH 终端
- [ ] 连接成功后通过 SSH 分配 PTY（伪终端），打开交互式 shell 会话
- [ ] 支持标准终端交互：命令输入、输出回显、翻页（Scrollback）
- [ ] 支持特殊键：Ctrl+C 中断、Tab 补全、方向键历史
- [ ] 终端 I/O 在独立线程中异步处理，不阻塞 UI 渲染
- [ ] 异常场景：SSH 会话意外断开时，状态栏显示"连接已断开"，并提供重连按钮
- [ ] 异常场景：网络波动导致 SSH 通道不稳定时，缓冲区处理不崩溃
- [ ] 边界情况：输出大量数据时（如 `cat` 大文件），终端不卡死，支持滚动查看

### US-003: SFTP 文件管理
- [ ] SSH 连接建立后自动通过同一会话打开 SFTP 通道
- [ ] 以表格形式展示远程文件列表：名称、大小、修改时间、权限
- [ ] 支持按列排序（点击表头）
- [ ] 支持新建文件夹、删除文件/文件夹
- [ ] 目录面包屑导航，点击可跳转
- [ ] 支持"返回上级目录"（..）
- [ ] 异常场景：SFTP 通道打开失败时，SFTP 面板显示错误信息，不影响 SSH 终端继续使用
- [ ] 异常场景：删除文件失败时（权限不足），提示"删除失败：权限不足"
- [ ] 边界情况：目录为空时，显示"此目录为空"

### US-004: 目录跟随
- [ ] 工具栏/面板上提供目录跟随开关（Toggle），默认开启
- [ ] 开启时：SSH 终端中检测到 `cd` 命令执行成功后，SFTP 面板自动切换到目标路径
- [ ] 关闭时：SSH 终端中的目录切换不影响 SFTP 面板
- [ ] 跟随实现应解析 Shell Prompt 中的当前路径（如 `user@host:/path$`），或通过 SSH 发送 `pwd` 命令获取
- [ ] 异常场景：SSH Prompt 自定义格式导致路径无法解析时，尝试 fallback 到 `pwd` 命令
- [ ] 边界情况：`cd` 到不存在的目录时，SFTP 不跟随，终端显示错误信息

### US-005: 多 Tab 管理
- [ ] 终端区域支持多个 Tab，每个 Tab 代表一个独立的 SSH 连接
- [ ] 每个 Tab 独立 keepalive 保活，互不影响
- [ ] 点击 Tab 切换激活的连接，SFTP 面板跟随当前激活 Tab 的会话
- [ ] 支持关闭 Tab（断开对应 SSH 连接）
- [ ] 支持新建 Tab（新建连接）
- [ ] Tab 显示连接名称或 host 简写
- [ ] 异常场景：某个 Tab 的连接断开，其他 Tab 不受影响
- [ ] 边界情况：同时打开 10+ 个 Tab 时，前端性能不显著下降

### US-006: 本地终端
- [ ] 支持切换到"本地模式"，打开本地 Shell（Windows 上默认为 PowerShell）
- [ ] 本地终端支持 CMD 和 PowerShell 切换
- [ ] 本地终端交互体验与 SSH 终端一致
- [ ] 异常场景：本地 Shell 进程崩溃时，自动重启并提示

### US-007: 文件传输（上传/下载）
- [ ] SFTP 面板提供上传按钮，点击后打开系统文件选择器，选中文件后上传到 SFTP 当前目录
- [ ] 支持选中远程文件后点击下载，保存到本地目录
- [ ] 传输过程中显示实时进度条：传输速度、已传输/总大小、剩余时间估算
- [ ] 传输面板/状态栏可查看当前进行中的传输任务列表
- [ ] 支持在传输过程中点击取消按钮停止传输
- [ ] 取消后自动清理已传输的残缺文件
- [ ] 上传/下载在独立后台线程中执行，不阻塞 SSH 终端交互及其他 UI 操作
- [ ] 支持同时进行多个传输任务（如同时上传和下载），各自独立互不阻塞
- [ ] 异常场景：下载/上传大文件（>1GB）时，进度条准确，不卡 UI
- [ ] 异常场景：传输中途网络断开，标记传输失败，不产生残文件
- [ ] 边界情况：上传空文件时正确处理，进度瞬间完成
- [x] 文件冲突处理：
  - 目标路径存在同名文件时，弹窗提示用户选择
  - 选项：覆盖 / 跳过 / 重命名
  - 提供"全部覆盖"/"全部跳过"快捷操作，避免重复弹窗

### US-008: 本地文件浏览器
- [ ] 切换到本地终端模式时，右侧面板从 SFTP 切换为本地文件浏览
- [ ] 显示本地文件系统目录结构，支持目录导航、排序
- [ ] 支持"在文件资源管理器中打开"（调用系统资源管理器打开当前目录）
- [ ] 不实现本地文件的查看、编辑、上传/下载等操作

### US-009: SSH 主机密钥验证
- [ ] 首次连接远程主机时，弹出主机密钥确认对话框，展示 key type、fingerprint（SHA256）
- [ ] 对话框提供"信任此主机"和"取消连接"两个操作
- [ ] 用户选择信任后，将主机密钥写入 known_hosts 文件（应用内 `known_hosts.json` 或系统 `~/.ssh/known_hosts`）
- [ ] 后续连接同一主机时，自动校验主机密钥是否匹配
- [ ] 密钥不匹配时弹窗警告，显示"主机密钥已变更"详情，并阻止连接
- [ ] 提供操作入口可从 known_hosts 中移除已信任的主机记录
- [ ] 异常场景：known_hosts 文件损坏或不可读时，按首次连接处理
- [ ] 边界情况：同一 host:port 对应多个密钥（如 IPv4/IPv6）时分别存储

### US-010: 断线完整重连
- [ ] SSH 连接断开后，用户可在当前 Tab 内点击"重连"按钮发起完整重连
- [ ] 重连流程：重新建立 TCP 连接 → SSH 认证 → 恢复 SFTP 通道（若开启）
- [ ] 重连使用已保存的连接配置，无需重新输入凭据
- [ ] 重连成功后，终端恢复可用状态，目录跟随开关保持断线前设置
- [ ] 重连失败时显示明确错误信息（区分网络不可达/认证失败/超时）
- [ ] 重连过程中状态栏显示"重连中…"，期间禁止用户输入
- [ ] 异常场景：原连接配置已被删除时，提示用户重新创建连接
- [ ] 边界情况：连续重试 3 次失败后停止自动重试，需用户手动触发

### US-011: 设置与外观定制
- [ ] 提供全局设置面板（标题栏齿轮图标），以弹窗或侧边面板呈现
- [ ] **终端设置**：
  - scrollback 行数（可选值：1000 / 5000 / 10000 / 50000，默认 5000）
  - 字体大小（10-24px，默认 14）
  - 字体族（Consolas / Cascadia Code / Fira Code / JetBrains Mono，含 fallback）
- [ ] **主题设置**：
  - 内置多套配色方案：暗色（默认）、亮色、经典终端绿、Solarized Dark
  - 切换后实时预览效果
  - 设置持久化到 localStorage，重启后保持
- [ ] **连接设置**：
  - 默认 SSH 端口（可修改，默认 22）
  - 连接超时时间（5s / 10s / 15s / 30s，默认 15s）
- [ ] 所有设置项有合理的默认值，首次使用无需配置
- [ ] 异常场景：设置文件损坏时，回退到默认值

## 4. 非功能需求 (Non-functional Requirements)

- **架构**：
  - Rust 后端基于 Tokio 异步运行时，使用 `russh` 作为 SSH 协议实现，`russh-sftp` 作为 SFTP 协议实现
  - 每个 SSH 连接对应一个独立的 Tokio task，终端 I/O、SFTP 操作、文件传输通过各自的 Channel 在同一 Session 上并发运行，无需额外线程同步
  - SSH 交互式 shell 通过 `request_pty` 分配 PTY，通过 `request_shell` 启动交互式 shell，基于 async event 机制推送输出到前端
  - 文件传输任务在独立 Tokio task 中执行，不阻塞 Tauri 主线程和前端 UI
  - 文件传输任务支持并发执行，传输间互不干扰
- **性能**：
  - SSH 连接建立时间 < 3s（局域网环境下 < 1s）
  - SFTP 目录列表加载 < 2s（1000 个文件以内）
  - 文件传输速度接近原生 SCP/SFTP 水平（减少 Rust 层开销）
  - 终端输入延迟 < 50ms
  - 应用启动时间 < 3s
- **安全**：
  - 凭据使用平台级加密存储（Windows DPAPI）
  - 不在日志中明文记录密码或密钥内容
  - 私钥文件路径不包含敏感信息
- **兼容性**：
  - 支持 Windows 10/11
  - 支持 OpenSSH 格式私钥（推荐），PuTTY .ppk 格式可选
  - 后端 SSH 兼容主流 Linux 发行版的 OpenSSH Server
- **可靠性**：
  - 单个 Tab 的连接断开不影响其他 Tab
  - SFTP 通道异常不影响 SSH 终端的正常使用
  - 应用异常退出时，不丢失已保存的连接配置

## 5. 功能边界 (Out of Scope)

- 暂不支持 SCP 协议（仅 SFTP）
- 暂不支持 SSH 端口转发/隧道
- 暂不支持本地文件上传/下载（本地模式下文件浏览器仅用于查看和定位）
- 暂不支持 macOS / Linux 原生编译（后续版本考虑）
- 暂不支持串口终端连接
- 暂不支持 SFTP 断点续传
- 暂不支持文件拖拽上传/下载
